Chapter 01

인공지능(AI)의 이해

인간의 지능과 인공지능의 차이를 이해하고, AI가 어떻게 작동하며 우리 생활에 어떻게 활용되는지 알아봅니다.

1.1 인공지능이란 무엇인가?

핵심 정의

인공지능(Artificial Intelligence, AI)이란 문제 해결, 추론, 예측, 생성 등과 같은 인간의 지능적 특성을 모방하기 위한 컴퓨터과학의 분야입니다.

인간은 지능을 활용하여 새로운 대상이나 상황에 맞게 적절한 판단과 의사 결정을 합니다. 이를 위해서는 문제 해결, 추론, 학습, 예측, 인식, 생성 등 다양한 세부적인 지능 능력이 필요합니다. 인공지능은 바로 이러한 인간의 지능적 특성을 컴퓨터가 모방할 수 있도록 연구하는 분야입니다.

1.2 인간의 지능과 인공지능의 차이

인간은 오감(시각, 청각, 촉각, 미각, 후각)으로 외부의 다양한 자극을 감지하고 인식하는 반면, 인공지능은 센서를 통해 대상을 감지합니다.

구분인간의 지능인공지능
감지 방식오감(시각·청각·촉각·미각·후각)센서(카메라, 마이크, 라이다 등)
판단 기반경험과 직관을 바탕으로 유연한 판단학습된 데이터와 알고리즘 기반 패턴 판단
강점감정, 공감, 창의성 등 복합적 사고대량 데이터 빠르게 처리, 반복 작업에 강점
에너지에너지 소모 적고 범용적 지능 보유특정 영역에서 인간 초월 성능 발휘 가능

1.3 튜링 테스트

🧪 튜링 테스트(Turing Test)란?

1950년 영국의 수학자 앨런 튜링(Alan Turing)이 제안한 테스트로, 컴퓨터가 인간과 구별할 수 없을 정도로 대화할 수 있는지를 평가하는 방법입니다.

사람이 컴퓨터와 대화를 나누면서 상대방이 사람인지 컴퓨터인지 구분하지 못한다면, 해당 컴퓨터는 '지능이 있다'고 판단합니다.

튜링 테스트는 인공지능 연구의 출발점이 된 중요한 개념입니다. 오늘날의 ChatGPT, Claude와 같은 대화형 AI가 등장하면서 이 질문은 더욱 현실적인 의미를 갖게 되었습니다.

1.4 인공지능의 활용 사례

인공지능은 우리 일상의 다양한 분야에서 활용되고 있습니다.

분야활용 내용
의료X-ray, CT 영상에서 질병을 자동으로 감지하고 진단을 보조
자율주행카메라·라이다 등 센서 데이터를 분석하여 스스로 주행 판단
음성 인식시리(Siri), 빅스비(Bixby) 등 음성 비서가 사용자 명령을 이해
추천 시스템넷플릭스, 유튜브 등이 사용자 취향에 맞는 콘텐츠를 추천
생성형 AIChatGPT, Claude 등이 텍스트·이미지·음악 등을 새롭게 생성
Chapter 02

데이터의 이해

인공지능이 학습하고 판단하기 위해서는 데이터가 필수적입니다. 데이터의 유형, 속성, 그리고 전처리 과정에 대해 알아봅니다.

2.1 데이터의 유형

정형 데이터와 비정형 데이터

구분정형 데이터 (Structured)비정형 데이터 (Unstructured)
형태행과 열로 구성된 표 형태일정한 형식이 없는 자유로운 형태
예시엑셀, 데이터베이스, 성적표, 출석부이미지, 동영상, 음성, SNS 게시글
특징검색과 분석이 비교적 용이분석을 위해 별도의 처리 기술 필요

2.2 데이터의 속성

데이터 속성은 숫자로 표현되는 수치형 데이터와 여러 범주 중 해당하는 값을 갖는 범주형 데이터로 분류됩니다. 수치형 데이터는 다시 연속형과 이산형으로 나뉩니다.

분류설명예시
수치형 — 연속형연속적으로 변하는 값키(165.3cm), 체온(36.5°), 시간
수치형 — 이산형셀 수 있는 정수 값학생 수(35명), 점수(85점)
범주형여러 범주 중 해당 값혈액형(A/B/O/AB), 성별, 학년

2.3 데이터 전처리

핵심 정의

데이터 전처리는 데이터 분석에 적합한 상태로 변환하는 과정입니다. 결측치, 이상치 등을 확인하고 처리 방법을 결정하며, 필요한 경우 정규화를 통해 속성값 간 분포를 조정합니다.

결측치 (Missing Value)데이터에 값이 존재하지 않는 것. 빈 칸이 있는 셀이 이에 해당합니다.
이상치 (Outlier)잘못 입력되었거나 정상 범위에서 많이 벗어난 데이터를 말합니다.
정규화 (Normalization)서로 다른 범위의 데이터를 일정한 범위(예: 0~1)로 변환하여 비교 가능하게 만드는 과정입니다.
💡 생각해보기

학교 급식 영양 데이터에서 칼로리가 '50,000 kcal'로 기록되어 있다면? 이것은 이상치일 가능성이 높습니다. 보통 한 끼 급식은 600~900 kcal 수준이기 때문입니다. 데이터를 그대로 사용하면 잘못된 분석 결과를 얻을 수 있어, 전처리가 꼭 필요합니다.

Chapter 03

기계학습의 원리

기계학습(Machine Learning)은 데이터를 기반으로 알고리즘이 패턴을 학습하고, 이를 통해 예측하거나 결정을 내리는 기술입니다.

🔍 인공지능 > 기계학습 > 딥러닝

인공지능(AI): 인간의 지능을 모방하는 컴퓨터과학의 가장 넓은 분야

기계학습(ML): 인공지능의 하위 분야로, 데이터를 기반으로 패턴을 학습하는 기술

딥러닝(DL): 기계학습의 하위 분야로, 인공신경망을 활용하는 기술

기계학습은 지도학습, 비지도학습, 강화학습으로 나뉘며, 각각의 방식은 학습 데이터와 목표에 따라 다르게 적용됩니다.

3.1 지도학습 (Supervised Learning)

지도학습은 인공지능에게 문제와 정답을 모두 제공하고 학습시키는 방식입니다. 주어진 데이터(입력값)와 그에 대응하는 정답(레이블)을 기반으로 알고리즘이 패턴을 학습합니다. 새로운 데이터가 주어졌을 때 학습된 패턴으로 예측하거나 분류합니다.

분류(Classification)와 회귀(Regression)

구분분류 (Classification)회귀 (Regression)
정의데이터를 특정 클래스(카테고리)로 구분연속적인 수치 값을 예측
예시이메일 → '스팸'/'정상', 이미지 → '고양이'/'개'주택 가격, 주식 변동, 내일 기온 예측
결과범주(카테고리)연속적인 숫자
📌 핵심 포인트

지도학습의 성능은 훈련 데이터의 품질과 양에 크게 영향을 받습니다. 정확한 레이블(정답)이 포함된 데이터셋이 반드시 필요합니다.

3.2 비지도학습 (Unsupervised Learning)

비지도학습은 인공지능에게 정답을 주지 않고, 데이터만 제공해서 스스로 숨겨진 패턴이나 구조를 찾도록 하는 학습 방식입니다.

군집화 (Clustering)

비슷한 특성을 가진 데이터를 같은 그룹으로 나누는 작업입니다. 예를 들어, 온라인 쇼핑몰에서 고객의 구매 기록을 분석해 비슷한 구매 습관을 가진 고객끼리 묶고, 맞춤형 추천이나 마케팅 전략을 세울 수 있습니다.

차원 축소 (Dimensionality Reduction)

복잡하고 많은 특징을 가진 데이터를 더 간단하게 만드는 작업입니다. 수많은 이미지 속에서 중요한 특징만 뽑아내어 시각화하거나, 분석 속도를 빠르게 합니다.

3.3 강화학습 (Reinforcement Learning)

강화학습은 인공지능이 스스로 시행착오를 통해 '보상'을 최대화하는 방향으로 학습하는 방식입니다. 보상(reward)과 불이익(penalty)을 반복 경험하면서 최선의 전략을 찾아갑니다.

강화학습의 핵심 구성 요소

에이전트 (Agent)학습을 수행하는 주체 (예: 게임 속 AI 캐릭터)
환경 (Environment)에이전트가 상호작용하는 공간 (예: 게임, 로봇 활동 공간)
상태 (State)현재 상황에 대한 정보
행동 (Action)에이전트가 선택할 수 있는 행동
보상 (Reward)특정 행동을 했을 때 받는 점수나 피드백

기계학습 3가지 유형 비교

구분지도학습비지도학습강화학습
정답 제공제공 (레이블 있음)미제공없음 (보상으로 학습)
학습 방식입력-정답 쌍데이터 패턴 분석시행착오
대표 기법분류, 회귀군집화, 차원 축소보상 최대화
활용스팸 필터, 이미지 인식고객 세분화, 추천게임 AI, 자율주행
Chapter 04

딥러닝의 원리

딥러닝은 기계학습의 하위 분야로, 인공신경망을 활용하여 데이터를 처리하고 복잡한 문제를 해결합니다.

4.1 딥러닝이란?

딥러닝(Deep Learning)은 인간의 뇌 신경망 구조를 본떠 만든 '인공신경망'을 이용하는 기술입니다. 다층 구조를 가진 신경망을 통해 이미지, 음성, 텍스트 등에서 높은 수준의 특징을 자동으로 추출합니다.

사람이 사과와 바나나를 구별할 때 빨간색이면 사과, 노란색이면 바나나라고 판단하듯이, 딥러닝도 수많은 사진을 보고 색깔과 모양 같은 특징을 스스로 찾아 구별하는 규칙을 배웁니다.

인공신경망의 기본 구조

입력층 (Input Layer)데이터의 특징을 입력받는 층. 예: 과일의 색깔, 크기, 모양
은닉층 (Hidden Layer)입력 정보를 분석하여 규칙과 패턴을 찾는 과정. 층이 많을수록 복잡한 패턴 학습 가능
출력층 (Output Layer)최종 판단 결과를 도출. 예: 사과인지 바나나인지 판별

4.2 인공신경망의 학습 과정

사과와 바나나를 구별하는 예시를 통해 학습 과정을 살펴봅시다.

초기 상태

무작위 가중치로 시작. 초록색 바나나를 사과로 착각하는 등 잘못된 예측 가능

순전파 (Forward Propagation)

입력층에 특징을 넣으면, 은닉층을 거쳐 출력층까지 전달. 현재 가중치 기반으로 예측

오차 계산

예측과 정답을 비교하여 오차 계산. 예: '바나나'인데 '사과'로 예측 → 오차 발생

역전파 (Backpropagation)

오차를 기반으로 가중치 조정. 연쇄 법칙(chain rule)으로 각 가중치의 영향도 계산

가중치 업데이트

조정된 값으로 가중치 업데이트. 다음 번에 더 정확한 예측 가능

반복 학습

위 과정 반복 → 점점 더 정확하게 판별 가능

4.3 CNN (Convolutional Neural Network)

핵심 정의

CNN은 사진(이미지)을 보고 분석하는 데 특화된 딥러닝 알고리즘입니다. 사람이 얼굴을 볼 때 눈→코→입 같은 특징을 차례로 보고 판단하듯이, CNN도 이미지의 부분적인 특징부터 전체를 이해합니다.

CNN의 핵심 아이디어

이미지를 한 번에 통째로 보는 것이 아니라, 작은 필터(창문)를 사용해서 부분적으로 살펴봅니다. 필터가 이미지 위를 움직이며 중요한 특징을 찾습니다.

CNN의 기본 구조

입력층

이미지를 입력 (예: 고양이 사진)

합성곱층 (Convolutional Layer)

필터가 이미지를 부분적으로 살펴보며 특징 추출

풀링층 (Pooling Layer)

중요한 정보만 추려 데이터 크기 축소

완전연결층 (Fully Connected Layer)

추출된 특징을 종합하여 최종 분류

4.4 GAN (Generative Adversarial Network)

핵심 정의

GAN은 이미지를 생성하는 데 특화된 딥러닝 알고리즘입니다. 두 개의 인공신경망(생성자와 판별자)이 경쟁하면서 함께 발전합니다.

생성자 (Generator)랜덤한 숫자에서 이미지를 만들어냄. 점점 더 정교하게 생성
판별자 (Discriminator)진짜와 가짜를 구별. 점점 더 정확하게 구분
🎭 GAN 학습 과정

① 생성자가 얼굴 사진 생성 → ② 판별자가 진짜 여부 판단 → ③ 처음에는 어색하지만 판별자가 지적 → ④ 생성자가 더 자연스럽게 수정 → ⑤ 반복 끝에 실제 같은 얼굴 생성 성공

4.5 Transformer

핵심 정의

Transformer는 문장처럼 순서가 있는 정보(텍스트)를 처리하는 데 특화된 딥러닝 모델입니다. 번역, 요약, 챗봇, 작문, 음악 생성 등에 활용됩니다.

핵심 원리: Attention (주의 메커니즘)

문장 안에서 어떤 단어가 서로 관련 있는지 스스로 판단해서 집중합니다.

💬 예시

"길동이는 비가 오는 날이면 항상 우산을 챙긴다."
AI는 '비가 오는 날'이라는 조건과 '우산을 챙긴다'는 행동 사이의 연결을 파악합니다.

Chapter 06

인공지능 윤리

인공지능이 인간과 사회에 미치는 영향을 바르게 조절하고, 책임감 있게 사용하기 위한 원칙과 기준을 알아봅니다.

6.1 인공지능 윤리란?

한 줄 정의

인공지능 윤리(AI Ethics)란, AI가 사람에게 해를 끼치지 않고, 공정하고 책임감 있게 사용되도록 기준을 세우는 것입니다.

AI는 사람처럼 판단하거나 결정을 내릴 수 있지만, 항상 공정하거나 옳은 것은 아닙니다. 사람을 차별하거나, 거짓 정보를 만들어내거나, 누군가에게 피해를 줄 수 있어 '어떻게 쓰이는가'가 매우 중요합니다.

⚠️ 생각해볼 질문들

Q. AI 채용 시스템이 여성보다 남성을 더 뽑는다면? → 학습 데이터의 성별 편향 때문일 수 있음

Q. 자율주행차 사고 시 누가 책임? → 제조사, 개발자, 탑승자 중 책임 소재 불명확

Q. 딥페이크로 누군가를 조작한다면? → 초상권과 명예를 심각하게 침해

6.2 데이터 편향성 (Bias in Data)

핵심 정의

데이터 편향성이란, AI가 학습하는 데이터가 특정 정보에만 집중되거나 일부만 반영되어 균형 잡히지 않은 상태입니다. AI가 골고루 배운 게 아니라 치우친 정보만 배운 것입니다.

인공지능은 받은 데이터를 그대로 믿고 학습합니다. 데이터에 문제가 있어도 스스로 알아내거나 고치지 못해 잘못된 판단이나 차별적 결과가 나올 수 있습니다.

편향성 발생 원인

데이터 양 부족/불균형

예: 질병 진단 데이터를 남성 위주로 수집 → 여성 환자에게 부정확한 진단

데이터 품질 저하

오래되거나 부정확한 정보 → AI가 오류까지 학습

사람의 선입견 반영

수집·정리 과정에서 특정 관점에 치우침 → AI도 편향된 판단

6.3 편향성을 줄이기 위한 노력

방법내용
양질의 데이터다양한 성별·나이·지역·배경 포함, 공공 데이터를 신뢰성 있게 가공
공정한 알고리즘특정 집단에 불리하지 않도록 설계, 차별 요인 사전 점검
비판적 수용AI 판단이 항상 정답은 아님, 데이터 기반 비판적 사고 필요

6.4 설명 가능한 인공지능 (XAI)

기존 AI는 어떻게 판단했는지 알 수 없는 '블랙박스'처럼 작동합니다. XAI(Explainable AI)는 AI 결정을 사람이 이해할 수 있도록 설명하는 기술입니다.

🔎 XAI 활용

의료: 왜 이 환자를 '고위험'으로 분류했는지 설명

금융: 왜 대출 신청을 거절했는지 근거 제시

제조업: 왜 이 제품을 '불량'으로 판정했는지 이유 제공

📌 핵심 정리

"데이터 편향성은 AI의 가장 중요한 윤리 문제. 좋은 데이터 + 공정한 설계 + 비판적 수용이 모두 필요합니다."
